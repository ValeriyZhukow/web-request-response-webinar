TCP/IP Model

Transmission Control Protocol / Internet Protocol

набір мережевих протоколів передачі даних чотири рівні стека - канальный (езернет, ..) - сетевой (IP) - транспортный (tcp, udp, ..) - прикладной IP & TCP - main protocols в 1970 ARPA мережі у світі могли бути об'єднані у формі віртуальної «мережі мереж» ARPAnet була на основі TCP/IP Сьогодні стек використовується для зв'язку комп'ютерів в Інтернет

TCP/IP Model: Application layer

прикладний на кожному рівні дані називаються по своєму потік (TCP) і повідомлення (UDP) на транспортному — сегмент і пакет взаємодія програм з інформацією програма-клієнт та програма-сервер поштові SMTP та POP3 передача файлів — FTP та TFTP емуляція віддаленого терміналу — TELNET HTTP, DNS

TCP/IP Model: Transport layer

транспортний транспортні послуги прикладним процесам TCP - Transmission Control Protocol забезпечує надійне доставляння байтових потоків UDP - User Datagram Protocol без установлення з'єднання не гарантує їхнього доставляння через порт

TCP/IP Model: Internet layer

міжмережевий айпи ? структура пакетов (мало смысла, лучше общий принцип) уникальный идентификатор устройства

канальний рівень доступу до середовища передачі обмен сигналами закодироваными между 2ма устройствами как передать? как прочитать?

OSI Model

Первая вещь, с которой сталкивается любой, кто изучает сети — это модель OSI семиуровневая эталонная модель Так, например, TCP — это протокол четвёртого уровня, и он сидит на шее IP — протоколе третьего уровня. Модель OSI была задумана ещё в 70-е годы позиционировалась как соперник стеку TCP/IP TCP и UDP едут верхом на IP который в свою очередь передвигается на Ethernet, PPP OSI помогает иллюстрировать зависимость одних протоколов от других the need for layered architecture data communication among heterogeneous systems - difficulties and solutions network complexities different types of hardware & software different operation systems different types of data to be transfered (text, images, music, video, ..) data must be transfered without errors many different paths may have to be taken yet Мережева модель OSI
базова еталонна модель взаємодії відкритих систем абстрактна мережева модель для комунікацій і розробки мережевих протоколів Open Systems Interconnection Basic Reference Model Представляє рівневий підхід до мережі Вважається, що модель занадто складна, а її реалізація займе занадто багато часу

OSI Model: Application Layer

  • Remote login to hosts: Telnet
  • File transfer: File Transfer Protocol (FTP), Trivial File Transfer Protocol (TFTP)
  • Electronic mail transport: Simple Mail Transfer Protocol (SMTP)
  • Networking support: Domain Name System (DNS)
  • ...

http: accessing resources over the web


                <protocol>://<server>/<path>

                communication protocol between the client and the server
                defines the address (uniform resource locator)

http: characteristics

request-response mechanism client sending a request to server server generates a response resource identification each http request includes a URI (uniform resource identifier) statelessness the server does not maintail any information about the transaction meta data support metadata about information can be exchanged in the message

request line - GET /index.html(relative url of the resource) HTTP/1.0 header lines - Host: www.site.com Date: 123 User-Agent: ... Accept-Language: .. carriage return/line feed message body (Content-Length: message-payload)

GET method means retreive whatever information whatever information is identified by the request-uri can Get static content and data produced by a program POST used to send data to the server for updates submit information to web server (posting to blog, submission of user form) information is included in message body the actual function depends on request uri
